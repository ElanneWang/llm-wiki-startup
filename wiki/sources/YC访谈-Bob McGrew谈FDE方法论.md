# YC 访谈：Bob McGrew 谈 FDE 方法论

## 摘要

Y Combinator 访谈 OpenAI 前首席研究官、Palantir FDE 模式早期架构师 Bob McGrew，深入讲解 FDE 模式的起源、运作方式和在 AI 创业中的应用。

## 关键信息

### 嘉宾
- **Bob McGrew**：OpenAI 前首席研究官，Palantir 早期员工，FDE 模式的架构师
- 过去一年为无数创业公司提供咨询，几乎所有公司都在研究 FDE 模式如何落地

### FDE 起源（补充现有知识）
- 源于 Palantir 服务美国情报机构的经历
- 客户挑战极其复杂、没有现成模板
- 起初被认为"无法规模化、太劳动密集"
- 如今被 AI Agent 公司奉为圭臬

### Echo & Delta 团队结构（新知识）
- **Echo**：行业洞察者，深入客户工作流程，挖掘 CEO 前五优先级中的核心痛点
- **Delta**：技术实干家，快速原型，优先速度而非完美抽象
- **总部核心产品团队**：将前线定制方案抽象为可复用的平台功能

### 与咨询的本质区别（关键补充）
- 咨询只解决一次性问题
- FDE 要求把一线经验反馈到产品中
- **反馈闭环** + **抽象能力** 是 FDE 的真正精髓
- 健康指标：核心产品是否在持续进化？交付效率是否在提高？

### AI 创业公司的应用
- AI Agent 市场过于碎片化，不存在"通吃型"产品
- 深度嵌入客户现场是唯一探索路径
- 每个企业部署都是一个独特的细分市场

### 商业模式
- 结果导向定价，而非按座席或 API 调用收费
- 关键杠杆：**产品杠杆**（同样的前线投入 → 更大合同 + 更低边际成本）
- 两个核心 KPI：交付价值 + 产品利用率

### 核心金句
> "做那些无法规模化的事"（doing things that don't scale at scale）

> "这不是捷径，而是深思熟虑的战略"

> "这是阵地战而非闪电战"

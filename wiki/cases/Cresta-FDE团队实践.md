# 案例：Cresta 的 FDE 团队实践

## 背景

- **公司**: Cresta（AI 客户服务/对话平台）
- **FDE 团队负责人**: Jove
- **团队规模**: 近百人
- **核心业务**: 为财富 100 强企业部署 AI 客户服务 Agent
- **独特优势**: 从 2015 年开始积累 Human-to-Human 对话数据

## 解决方案

### 团队结构
- **FDE**（前置部署 CTO）：技术实现、AI Best Practices、SDK 工具化、测试体系
- **FDPM**（前置部署 CEO）：客户关系、需求把控、信任建立、Quality 负责、Upsell

**配比**：1 FDPM : 2-3 FDE

### 人才画像
- 必须是资深工程师，不招 Junior
- 有 AI Agent 开发经验，2-3 周内能进项目
- 过硬的 Customer Facing 能力
- 面**对不完美世界**的韧性
- 多任务并行（同时 2-3 个项目）

### 方法论
- **数据驱动 Use Case 选择**：优先量大 + SOP 清晰 + 不需要复杂人工判断的场景
- **工作节奏**：
  - 前期：飞客户现场 2-3 天闭门会，定 Goal/KPI，验证 API
  - 中期：远程开发，每周/每天同步会
  - 后期：UAT 回访，指标达标后退出
- **商务模式**：倒贴 2-3 个月做 PoC，"试吃策略"

## 结果

- 成功服务财富 100 强企业
- FDE/FDPM 双角色结构实现高效协作
- 雪球效应：行业专家按领域分组，新 FDE 直接复用前面积累的经验

## 关键洞察

1. **FDPM 是 FDE 模式的进化**：Palantir 的 Echo/Delta 在实践中演化为 FDPM/FDE，更适应当前 AI 时代
2. **不招 Junior**：FDE 门槛极高，因为需要独当一面直面客户 CTO
3. **工具化是未来**：工具让 FDE 更高效，远期 FDE 将分化为高端/低端两个市场
4. **雪球效应**：行业知识库的积累是新 FDE 的核心竞争力
